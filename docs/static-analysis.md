# Статический анализ

В проекте используется статический анализ для раннего обнаружения ошибок,
нарушений стиля и потенциальных проблем безопасности.

## Ruff

`Ruff` проверяет стиль, порядок импортов и распространенные ошибки Python-кода.

Запуск:

```bash
ruff check .
```

Ожидаемый результат:

```text
All checks passed!
```

## Bandit

`Bandit` выполняет SAST-анализ Python-кода и ищет небезопасные конструкции:
использование опасных функций, слабую криптографию, небезопасную работу с
внешними данными и другие типовые проблемы.

Запуск:

```bash
bandit -c pyproject.toml -r src
```

Ожидаемый результат:

```text
No issues identified.
```

## CI

Статический анализ автоматически выполняется в GitHub Actions при push в ветки
`main`, `develop` и при pull request.

Файл конфигурации:

```text
.github/workflows/ci.yml
```

В CI выполняются:

- установка Python 3.12;
- установка зависимостей проекта;
- `ruff check .`;
- `pytest`;
- `bandit -c pyproject.toml -r src`;
- `pip-audit`.
