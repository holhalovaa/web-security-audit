# Использование AI-инструментов

В ходе разработки проекта применялся AI-ассистент как вспомогательный инструмент
для анализа, ревью и улучшения качества программного продукта. Итоговые решения,
структура проекта и проверка работоспособности контролировались разработчиком.

## Где использовался AI-ассистент

| Направление | Что было сделано |
| --- | --- |
| Анализ требований | Сверка проекта с вариантом 8 и обязательными компонентами курсовой работы |
| Архитектура | Проверка разделения ответственности между CLI, краулером, парсером, проверками и отчетами |
| Code review | Поиск слабых мест в проверках security headers, тестах и документации |
| Тестирование | Расширение тестов для активных сканеров, краулера, CLI, HTTP-клиента и отчетов |
| Документация | Подготовка README, API-документации, архитектурного описания и вспомогательных документов |
| Безопасность | Проверка наличия SAST, dependency audit и безопасного режима запуска |

## Принципы использования

- AI использовался как ассистент, а не как единственный источник решений.
- Изменения проверялись автоматическими тестами и статическим анализом.
- В проект не добавлялись фиктивные проверки: XSS, SQLi, CSRF и headers
  реализованы как реальные анализаторы поведения страниц и форм.
- Активные проверки задокументированы как потенциально воздействующие на цель,
  поэтому их рекомендуется запускать только на разрешенных стендах.

## Проверенные результаты

После доработок были выполнены:

```bash
pytest
ruff check .
bandit -c pyproject.toml -r src
pip-audit
```

На момент проверки тесты проходили успешно, покрытие было выше 90%, статический
анализ не находил проблем, а аудит зависимостей не выявлял известных
уязвимостей.

## Роль AI в курсовом проекте

Использование AI-инструментов помогло ускорить ревью, улучшить полноту
документации, повысить покрытие тестами и привести проект к более аккуратному
production-grade виду. При этом проект остается прозрачным: вся логика находится
в исходном коде, покрыта тестами и воспроизводимо запускается локально или в
Docker.
