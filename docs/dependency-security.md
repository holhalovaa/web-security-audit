# Проверка зависимостей

Для контроля безопасности сторонних библиотек используется `pip-audit`.
Инструмент сравнивает установленные Python-пакеты с базами известных
уязвимостей и сообщает, если для зависимости есть CVE или рекомендации по
обновлению.

## Используемые зависимости

Основные runtime-зависимости:

- `requests` — HTTP-запросы;
- `beautifulsoup4` — HTML-парсинг;
- `playwright` — PDF-рендеринг отчета.

Dev-зависимости:

- `pytest`;
- `pytest-cov`;
- `ruff`;
- `bandit`;
- `pip-audit`.

## Запуск проверки

```bash
pip-audit
```

На Windows при проблеме с кодировкой вывода можно использовать:

```powershell
$env:PYTHONUTF8='1'
python -m pip_audit
```

Ожидаемый результат для актуального состояния проекта:

```text
No known vulnerabilities found
```

Локальный пакет `web-security-audit` может быть пропущен аудитом, потому что он
не опубликован на PyPI. Это нормально: `pip-audit` проверяет внешние зависимости,
а код проекта дополнительно анализируется через `Bandit`.
